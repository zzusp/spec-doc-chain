# 分析已有项目并生成文档

## 何时使用

- 用户要求「分析当前项目/这个仓库」「根据现有代码生成分析报告、PRD、技术设计」。
- 用户指定某个目录或仓库路径，要求基于其实现反推需求与设计文档。

## 目标输出

1. **分析报告**：基于代码与仓库结构得出的现状、范围、结论与建议。
2. **PRD**：从系统能力与代码行为反推的产品需求与功能列表。
3. **系统技术设计**：对现有架构、模块、接口、技术栈的归纳与设计说明。
4. （可选）**验收清单**与**澄清文档**：若用户需要完整文档链，可继续按 generate-doc-chain 生成。

## 路径与模板

- 输出路径**默认**遵守 **spec/<YYYY-MM-DD>/<需求名称>/**（与 spec-agent 一致），文件名使用 `01-analysis.md`、`02-prd.md`、`03-tech.md`、`04-acceptance.md`、`00-clarifications.md`。若用户未提供需求名称，可从仓库名或项目根目录名推导并确认。**多需求时**：目标目录按 reference/multi-requirement.md 解析；生成完成后可写入 spec/.active 为该需求路径。
- 各文档须按插件内 `templates/01-analysis.md`、`02-prd.md`、`03-tech.md`、`04-acceptance.md`、`00-clarifications.md` 框架填写，保持章节结构一致；每份文档含 **## 修订记录** 表，更新时追加一行。
- 若存在 **spec/00-global-memory.md**，撰写前须读取并作为全局约束纳入。

## 执行步骤

1. **确认范围**：确定要分析的代码根目录（默认当前工作区根目录或用户指定路径）。
2. **扫描与归纳**：
   - 目录结构、主要模块/包/服务划分。
   - 依赖与技术栈（package.json、requirements.txt、go.mod、pom.xml 等）。
   - 入口、配置、关键业务逻辑文件（可抽样阅读）。
   - 接口定义（API 路由、RPC、消息等，若有）。
   - 文档与注释中已有的需求或设计描述（README、设计文档等）。
3. **编写分析报告**：
   - 使用 `templates/01-analysis.md` 框架（含 原始需求、需求上下文采集、项目现状与相关模块、候选模块扫描、数据库现状、需求覆盖矩阵、需求满足性分析、风险与影响、结论、澄清补充、修订记录）。
   - 背景与目标：说明「对已有项目进行逆向分析与文档化」。
   - 现状分析：基于扫描结果写业务/技术现状、模块职责、约束。
   - 范围与边界：本分析覆盖的代码范围与排除范围。
   - 结论与建议：现有能力归纳、缺失或待改进点、后续 PRD/设计的建议。
4. **编写 PRD**：
   - 使用 `templates/02-prd.md` 框架。
   - 从现有功能与代码行为反推产品目标、用户与场景、功能列表与优先级。
   - 标注「由现有系统逆向归纳」，与分析报告结论对齐。
5. **编写系统技术设计**：
   - 使用 `templates/03-tech.md` 框架。
   - 总体架构、模块划分、技术选型、接口与数据等，基于扫描结果归纳而非臆造。
   - 与 PRD 中的功能对应，便于后续验收清单追溯。
6. **待澄清项**：若分析过程中存在无法从代码确定的二义性（如业务规则、优先级），创建或更新 `00-clarifications.md`（若不存在则按模板先创建），插入待澄清项；或先写入各文档的「澄清补充」区块，再同步到澄清文档。
7. **可选**：若用户需要完整文档链，在同一需求目录下继续生成 `04-acceptance.md`，并完善澄清文档。若 `00-clarifications.md` 已由步骤6 创建并写入待澄清项，此处仅补充从 04 提取的待澄清项（若有），**不覆盖**已有澄清内容；若 00 尚未存在，按 generate-doc-chain 规则创建并写入。

## 注意

- 分析结论应基于**实际代码与结构**，避免凭空编造需求或架构。
- 无法从代码推断的内容（如业务背景、优先级、非功能需求）应在文档中标注「待业务确认」或写入澄清文档。
- 撰写 01、02、03 时，**写回前须 whole-document review**（整体回顾该份文档，有矛盾可自行收敛则修正，需用户决策则写入澄清文档）。
- 与 generate-doc-chain 配合时，本技能产出相当于「分析报告 + PRD + 技术设计」已生成，后续仅需生成验收清单与完善澄清文档。
