# 分析已有项目并生成文档

执行以下流程：

1. 确认要分析的代码根目录（默认当前工作区根目录）及**需求名称**（用于输出路径）。
2. 调用 **analyze-existing-project** 技能：
   - 扫描目录结构、依赖、技术栈、关键模块与接口。
   - 在 **spec/<YYYY-MM-DD>/<需求名称>/** 下按模板生成：
     - `01-analysis.md`（基于现状的分析与结论）
     - `02-prd.md`（反推的产品需求）
     - `03-tech.md`（归纳的架构与设计）
     - `00-clarifications.md`（若有待澄清点则写入）
   - 若存在 `spec/00-global-memory.md`，撰写前读取并作为全局约束。
3. 若用户需要完整文档链，在同一目录下继续生成 `04-acceptance.md` 并同步澄清文档。
4. 回复：已生成的文件路径、分析结论摘要，以及建议的下一步（如补充澄清结论后同步回写）。

若用户提供了需求概要或业务背景，将其作为分析报告与 PRD 的补充输入。
