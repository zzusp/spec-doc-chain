# 澄清文档双向同步

执行以下流程：

1. 判断用户需求是「提取待澄清项」还是「根据澄清结论回写」或「两者都做」。
2. **提取待澄清项**：对指定或最近变更的文档执行 sync-clarification 方向一，将需用户澄清的内容写入澄清文档。
3. **澄清回写**：对澄清文档中状态为「已确认」且已填写结论的项执行 sync-clarification 方向二，更新对应文档（含 **whole-document review**，该条在澄清文档中保持已确认状态）。
4. **回写后必做**：澄清内容写回各文档后，**更新后的文档可能产生新的待澄清内容**。须对刚被回写更新的文档（以及若有级联则对级联后更新的每份文档）执行 **sync-clarification 方向一**，将新待澄清项写入澄清文档。
5. **回写后级联**：若回写导致其它文档需联动（如回写更新了 02-prd，则 03、04 可能需同步），须再调用 **cascade-update** 技能更新后置文档（该技能内部会对每份更新后的文档执行澄清提取）。回写后无论是否级联，都须完成上一步「对新内容的澄清提取」。
6. 回复：同步了哪些文件、澄清文档中新增/已确认的项、**本次新产生的待澄清项**（如有），以及建议的下一步（推动澄清循环）。**兜底**：本轮回写+级联+澄清提取执行一轮即结束，新待澄清项由用户填写后由用户显式发起下一轮，避免死循环。
