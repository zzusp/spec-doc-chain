# 全局记忆（示例 / 参考）

本文档为 **spec/00-global-memory.md** 的参考内容。该文件位于 **spec 根目录**（非各需求目录内），记录项目级上下文与用户/团队约定，供各需求文档（01/02/03/04）撰写时引用。

- 若 **spec/00-global-memory.md 不存在**：首次在 spec 下生成文档时，可创建该文件并写入下述最小模板，再读取作为全局约束。
- **禁止**将单次需求的范围、结论或描述写入全局记忆；此类内容属于需求文档或澄清文档。

## 最小模板（创建时可使用）

```markdown
# 全局记忆
本文档记录项目级上下文与用户/团队约定，供各需求文档引用。

## 项目级情况
- 待补充：技术栈、部署环境、合规要求等。

## 约定索引
> 约定索引用于去重与追踪来源。同一条约定在多个需求中被重复沉淀时，应复用同一 GM-ID，并在「来源需求」列中追加需求名称，而不是新增多行相似约定。

| GM-ID | 一句话约定                            | 适用范围      | 来源需求 |
|-------|---------------------------------------|---------------|----------|
| GM-001| 待补充：约定内容示例                 | 全局          | <需求名称> |

## 用户/团队约定
- 待补充：命名规范、交付口径等。可对应上方约定索引中的「一句话约定」逐条展开说明（如适用场景、背景、例外情况等）。
```
