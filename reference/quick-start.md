# 5 分钟上手指南

本页帮助你在单需求与多需求场景下快速跑通文档链与交付闭环。

## 单需求最简路径

1. **确定需求目录**  
   文档默认放在 `spec/<YYYY-MM-DD>/<需求名称>/`（例如 `spec/2025-03-11/user-center-refactor/`）。若未提供需求名称或日期，对话中会向你确认。

2. **生成文档链（二选一）**  
   - **从零生成**：在对话中说「从分析报告开始，按顺序生成整套文档」，或使用命令 `doc-chain-generate-all`。  
   - **基于现有代码**：说「分析当前项目，生成分析报告、PRD 和技术设计」，或使用命令 `doc-chain-analyze-project`。

3. **澄清与回写**  
   生成或更新后，若有待确认项会写入 `00-clarifications.md`。在澄清文档中填写「用户确认/补充」或「解决方案」，并把状态改为「已确认」，然后说「把澄清结论写回 PRD」（或对应文档），或使用命令 `doc-chain-sync-clarification`。可重复「填写澄清 → 回写」直到满意。

4. **交付（可选）**  
   当 PRD、技术设计、验收清单就绪后，说「按验收清单开始交付」或「跑验收直到全部通过」，或使用命令 `spec-delivery-run`。流程会按 A-xxx 开发 → 验收 → 不通过则修复 → 再验收，直至全部通过并写入交付日志。

## 多需求时

- **列出需求**：说「列出所有需求」或使用命令 `doc-chain-list`，查看 `spec/` 下所有需求目录及当前激活需求。  
- **切换当前需求**：说「切换到 xxx」或「当前需求设为 2025-03-11/req-b」，或使用命令 `doc-chain-set-active`。  
- 之后进行生成、级联、澄清、交付等操作时，未显式指定需求则会针对当前激活需求；也可在对话中直接指定需求名称。

## 常用入口一览

| 你想做的事           | 在对话中说（示例）                         |
|----------------------|--------------------------------------------|
| 生成整套文档         | 「从分析报告开始生成整套文档」             |
| 分析代码生成文档     | 「分析当前项目，生成分析报告、PRD、技术设计」 |
| 改了 PRD 要同步后面 | 「我改了 PRD，请同步技术设计和验收清单」   |
| 澄清写回文档         | 「把澄清文档里第 2 条的结论写回 PRD」      |
| 开始交付             | 「按验收清单开始开发，跑验收直到全部通过」 |

更多路径与模板约定见 [path-convention.md](path-convention.md)、[doc-types-and-order.md](doc-types-and-order.md)；流程衔接见 [flow-overview.md](flow-overview.md)。
